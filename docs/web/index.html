System Overview

SuperMean is a self-improving AI agent system designed for domain-specific problem solving. The system leverages multiple AI models and agents to provide comprehensive solutions across various domains.

Core Components

  • SuperAgent: The main orchestrator of the system
  • Planner: Responsible for task decomposition and execution planning
  • Medical Agent: Specialized agent for healthcare domain tasks
  • Model Router: Directs queries to appropriate AI models
  • Skill Library: Collection of reusable capabilities

Installation Guide

System Requirements

  • Operating System: Linux (Ubuntu 20.04+), macOS (10.15+), Windows 10/11
  • Python: 3.9 or higher
  • RAM: 16GB minimum, 32GB recommended
  • Storage: 10GB minimum for installation, 50GB+ recommended for operation
  • GPU: NVIDIA GPU with CUDA support (optional, for local model inference)

Installation Steps

  1. Clone the Repository

    git clone https://github.com/yourusername/supermean.git
cd supermean

  2. Set Up Virtual Environment

    python -m venv venv
# On Windows
venv\Scripts\activate
# On macOS/Linux
source venv/bin/activate

  3. Install Dependencies

    pip install -r requirements.txt

  4. Configure Environment Variables

    # Create a .env file in the root directory
touch .env

# Add the following variables to the .env file
API_KEY=your_api_key
MODEL_ENDPOINT=https://api.example.com/v1
MEMORY_STORAGE=local  # or 'cloud'

  5. Initialize the System

    python SuperMean/backend/run.py --init

  6. Start the Backend Server

    python SuperMean/backend/run.py

  7. Start the Frontend (in a new terminal)

    cd SuperMean/frontend
npm install
npm run dev

Troubleshooting

Common Issues

Model Router Connection Failures

If you experience connection issues with the Model Router:

  1. Verify your API keys are correctly set in the .env file
  2. Check network connectivity to the model endpoints
  3. Ensure the model service is available and not experiencing downtime
  4. Check the logs at logs/model_router.log for specific error messages
Memory System Errors

If the memory system fails to initialize or store data:

  1. Verify that FAISS is properly installed
  2. Check disk space availability for vector storage
  3. Ensure the memory directory has proper write permissions
  4. Try clearing the cache: python tools/clear_memory_cache.py
Agent Initialization Failures

If agents fail to initialize properly:

  1. Check that all required dependencies are installed
  2. Verify that the Model Router is running and accessible
  3. Ensure the agent configuration files are properly formatted
  4. Look for specific error messages in logs/agent_init.log
